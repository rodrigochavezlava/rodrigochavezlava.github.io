Suricata.yml¶

Suricata es una herramienta de vigilancia de red que se enfoca en el tráfico de red y en el envío de alertas. Las configuraciones aplicadas a Suricata son un poco diferentes, ya que su objetivo principal es analizar y detectar eventos en la red, sin necesidad de recopilar datos.

En las configuraciones de Suricata se pueden observar reglas relacionadas con el tráfico de Internet, validación de usuarios, registro de la actividad y especificaciones detalladas para la captura de paquetes, interfaces de red utilizadas para la captura, registro de eventos HTTP y un cuidado especial en la gestión de claves de cifrado.

Al final del archivo, se especifica qué registros está observando Suricata, es decir, qué eventos o actividades está monitoreando en la red.

types:
        - alert              # Tipos de eventos EVE a registrar (alertas)
        - http               # Tipos de eventos EVE a registrar (HTTP)
        - dns                # Tipos de eventos EVE a registrar (DNS)
        - tls                # Tipos de eventos EVE a registrar (TLS)

%YAML 1.1
---
# Configuración de Suricata

default-log-dir: /var/log/suricata/

# Definición de las reglas
rule-files:
  - /var/lib/suricata/rules/*.rules

# Configuración de los registros
outputs:
  - fast:  # Configuración de la salida rápida
      enabled: yes  # Habilitar la salida rápida
      filename: fast.log  # Nombre del archivo de salida rápida
      append: yes  # Agregar al archivo de salida rápida existente si existe
  - eve-log:  # Configuración de la salida de registros EVE
      enabled: yes  # Habilitar la salida de registros EVE
      filetype: regular  # Tipo de archivo de salida de registros EVE
      filename: eve.json  # Nombre del archivo de salida de registros EVE
      append: yes  # Agregar al archivo de salida de registros EVE existente si existe
  - syslog:  # Configuración de la salida de registros syslog
      enabled: yes  # Habilitar la salida de registros syslog

# Configuración de la detección de anomalías
anomaly:
  detection-ports:
    - 21  # Puerto FTP
    - 22  # Puerto SSH
    - 23  # Puerto Telnet
    - 25  # Puerto SMTP
    - 80  # Puerto HTTP

# Configuración de los protocolos de capa de aplicación
app-layer:
  protocols:
    dcerpc:
      enabled: yes
    smb:
      enabled: yes
    ftp:
      enabled: yes
    ssh:
      enabled: yes
    smtp:
      enabled: yes
    dns:
      enabled: yes
    modbus:
      enabled: no
    http:
      enabled: yes
    tls:
      enabled: yes
    enip:
      enabled: yes
    dnp3:
      enabled: yes
    nfs:
      enabled: yes
    ntp:
      enabled: yes
    tftp:
      enabled: yes
    ikev2:
      enabled: yes
    krb5:
      enabled: yes
    dhcp:
      enabled: yes
    snmp:
      enabled: yes
    sip:
      enabled: yes
    rfb:
      enabled: yes
    mqtt:
      enabled: yes
    rdp:
      enabled: yes
    http2:
      enabled: yes
    imap:
      enabled: yes

# Configuración de las interfaces
af-packet:
  - interface: enp0s3  # Nombre de la interfaz de red a utilizar
    cluster-id: 99  # ID del clúster
    cluster-type: cluster_flow  # Tipo de clúster
    defrag: yes  # Habilitar la desfragmentación de paquetes

# Configuración de la preprocesamiento
prelude:
  enabled: yes  # Habilitar la integración con Prelude
  log-dir: /var/log/suricata/  # Directorio de registro de Prelude

# Configuración de la extracción de archivos
file-extraction:
  enabled: yes  # Habilitar la extracción de archivos
  directory: /var/log/suricata/files  # Directorio donde se almacenan los archivos extraídos

# Configuración de los registros HTTP
http-log:
  enabled: yes  # Habilitar el registro de eventos HTTP
  filetype: regular  # Tipo de archivo de salida de registros HTTP
  filename: http.log  # Nombre del archivo de salida de registros HTTP
  append: yes  # Agregar al archivo de salida de registros HTTP existente si existe
  extended: yes  # Habilitar registros HTTP extendidos

# Configuración de los registros TLS
tls-log:
  enabled: yes  # Habilitar el registro de eventos TLS
  filetype: regular  # Tipo de archivo de salida de registros TLS
  filename: tls.log  # Nombre del archivo de salida de registros TLS
  append: yes  # Agregar al archivo de salida de registros TLS existente si existe
  extended: yes  # Habilitar registros TLS extendidos
