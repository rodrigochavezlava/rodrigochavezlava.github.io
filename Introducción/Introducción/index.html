Introducción¶

El proyecto consiste en llevar a cabo la instalación de un SIEM mediante la ejecución de un solo script. Esto se conseguirá mediante Docker Compose, ya que de esta forma crearemos en un solo archivo un contenedor para cada servicio necesario. En el mismo archivo se configurarán las conexiones entre sí de los softwares, así como la ruta de utilidad y cualquier tipo de configuración necesaria de cada software aislado.

Por esto, el proyecto constará de un archivo Docker Compose con lo mencionado anteriormente y diferentes archivos yml que configuran particularidades de los servicios.

Además, durante el proceso se hará uso de distintos archivos de configuración .rule para Suricata.

Todo esto con la finalidad de facilitar y automatizar el proceso de instalación de un SIEM completamente funcional y personalizable, con todas las facilidades posibles para el usuario final, incluyendo además un manual de usuario para que sea fácil el uso y entendimiento del script.
